‚öôÔ∏è How the Generator Engine Works

1Ô∏è‚É£ User Selects State + Situation

User lands on a page (e.g., /unpaid-wages-california/) or uses dropdowns to select their state and situation type. The page URL determines which modules to load.

2Ô∏è‚É£ Engine Loads Relevant Modules

State Module (e.g., california.js): Loads CA Labor Code ¬ß¬ß, DLSE procedures, CA-specific penalties, filing deadlines, court links.
Situation Module (e.g., unpaid-wages.js): Loads wage claim template, legal elements, calculation formulas, demand structure.
Industry Module (optional): Loads industry-specific terms if user selects one (e.g., SaaS, Construction).

3Ô∏è‚É£ Live Preview with Scroll-to-View

Split-pane UI: Form inputs on LEFT, live document preview on RIGHT.
On every keystroke: Preview updates instantly. When user focuses a field, preview scrolls to that section and highlights it with animation.
Uses trackFieldEdit() pattern from CONTRACT-GENERATOR-STANDARDS.md.

4Ô∏è‚É£ Full Edit Mode + Inline Editing

Users can click ANY highlighted text in preview to edit inline. Toggle "Full Edit Mode" to freely modify entire document (add/remove paragraphs). System warns before overwriting custom edits if form inputs change.

5Ô∏è‚É£ Export: Print, Copy, Download DOCX

User exports finalized letter. Options: Print (opens dialog), Copy to Clipboard (formatted text), Download as .docx file. Letter includes proper legal formatting, state-specific citations, and correct demand structure.
